# Описание телеграмм бота

## Технологии

* aiogram
* sqlite
* FSM
* csv
* request
* logging in folder and terminal

## Какое разделение должно быть

* пользователи
* админка

## Функциональные требования

### Общие требования

Бот должен корректно принимать из сайта информацию от заявок. Код сайта находится в папке site/.

### Пользовательский роутеры

* get_image - получить все свадебные фотографии (бот будет присылать список ссылок с яндекс диска, если с админки эти ссылки прикрепили)
* send_image - выслать фотографии в общий альбом
* plan - спрашивает пользователя, какую информацию о каком свадебном дне нужно получить (расписание 1-ого дня, расписание 2-ого дня)
* send_wishes - отправить пожелание к организационным моментам
* drinks - выбрать напитки, для пожелания
Быстрая навигация: команды /start, /menu, /plan, /gallery, /rsvp, /ask.

### Админская часть
* get_members - получить список всех участников, оставивших анкету
* get_all_members - получить список всех людей и людей из пары
* send_notification - отправить уведомление всем пользователям (чтобы можно было во время праздника уведомлять участников)
* add_image - добавляет фотографию на сайт
* add_member - добавить участника
* delete_memeber - удалить участника
* get_config - получаем конфиг сайта целиком
* update_cnfig - отправляет на сайт новый конфиг
Составление шаблонных писем и хранение их в БД, чтобы до свадьбы подготовить заготовки, а ужее во время свадьбы отправлять рассылку пользователям

Так же нужно продумать выводы в формате .csv, чтобы можно было анализировать количество участников и их предпочтения. И чтобы общение было в виде чата с пользователем и было вовлечение.

Проект должен быть модульным, не городи все в одной функции
